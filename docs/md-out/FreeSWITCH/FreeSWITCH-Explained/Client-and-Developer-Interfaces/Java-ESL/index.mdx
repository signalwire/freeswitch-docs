# Java Connection Options

There are at least three options for using Java to connect to the FreeSWITCH [Event Socket](../Event-Socket-Library/index.mdx#-new-).

### esl.jar (freeswitch core)

* License: Same as FreeSWITCH
* Available: esl.jar can be built as part of FreeSWITCH code base
* Features:  
   * Automatically generated by SWIG from core C ESL interfaces (so by definition, always up to date with core ESL).  
   * Requires the native esl\_lib dependencies to available at runtime.  
   * An example is provided at bottom of this page [Event Socket Library](../Event-Socket-Library/index.mdx#-new-).

### org.freeswitch.esl.client (freeswitch-contrib)

* License: Apache v2
* Available: In freeswitch-contrib git repository, see [Java ESL Client](Java-ESL-Client_7144076.mdx#about) for details and usage.
* Features:  
   * Inbound and outbound  
   * Handles event bodies  
   * Built using maven, and OSGi ready  
   * Runtime dependencies only on the excellent [Netty](http://www.jboss.org/netty) (Apache License) and [SLF4j](http://www.slf4j.org) (MIT License) libraries

Now this Java ESL Client is available and update on github <https://github.com/esl-client/> , welcome you to join us. ^\_^

### fs_eslib (Versafon)

* License: GPL v3
* Available: <http://versafon.com/versafonweb/Software.jsp>
* Type: Inbound
* Features: see the web page above.
