
# Provider nexVortex (USA) 

 Created by  Mario G, last modified on 2014.05.31

## [nexVortex.com](http://nexvortex.com)

Last Update: 17 January 2011

Sample external profile:

```xml
<include>
   <gateway name="nexvortex">
      <param name="username" value="your user name"/>
      <param name="password" value="your password"/>
      <param name="from-user" value="Put Your Caller ID Number here!" /> 
      <param name="proxy" value="px1.nexvortex.com"/>
      <param name="realm" value="px1.nexvortex.com"/>
      <param name="expire-seconds" value="3600"/>
      <param name="register" value="true"/>
      <param name="ping" value="600"/>
      <param name="caller-id-type" value="pid"/>
   </gateway>
</include>
```

NexVortex does not use the Remote-Party-ID, hence the param name = "from-user".

When you use the "from-user", this changes your user registration so that the DID is the registered user, so you must set the DID routing to point to the DID instead of the NexVortex username in the NexVortex call routing.
