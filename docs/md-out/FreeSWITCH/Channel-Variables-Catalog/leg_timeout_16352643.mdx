## leg_timeout

integer Timeout for each leg in an originate dialstring. Can be used in per-leg `[]`, but not in global `{}` for the dialstring. For global, use [originate\_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#originate_timeout).

You can also use [leg\_progress\_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#leg_progress_timeout) to specify the maximum time we will wait before we get media (whether it's early media, ringing or answer), allowing you to avoid going to voicemail for a particular line.

If you are using group confirm then you can cancel the timeout by using the [group\_confirm\_cancel\_timeout](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#group_confirm_cancel_timeout) channel variable. If [leg\_delay\_start](../FreeSWITCH-Explained/Dialplan/Variables-Master-List_16352570.mdx#leg_delay_start) is also used, `leg_timeout` will not start the timeout counter until after the extension starts to be bridged to.

**Usage**

```xml
<action application="bridge" data="[leg_timeout=15]user/hastoanswerquickly/some.domain.com,[leg_timeout=60]user/hasaminutetoanswer@some.domain.com"/>
```
