
# Glossary 

 

## About

This FreeSWITCH glossary is designed to demystify key technical terms related to telephony and specifically to FreeSWITCH. The terms can sometimes be confusing, and trip up understanding; hopefully this glossary will help.

Covered terms range from basic telephony elements like call legs and channels to more complex technologies such as CODECs and VoIP (Voice over IP). Understanding these concepts is essential to deepen understanding of telephony technologies and utilize FreeSWITCH more effectively.

## Call switching

See Wikipedia entries [telephone exchange](https://en.wikipedia.org/wiki/Telephone%5Fexchange) and [telephone switchboard](https://en.wikipedia.org/wiki/Telephone%5Fswitchboard).

## Call leg

A **call leg** is an inbound or outbound connection between the phone switch and an external party. 

See [Call Legs](./Call-Legs#0-about) page.

## Channel

In FreeSWITCH, a **channel** represents a _call leg_.

See [Call Legs](./Call-Legs.mdx#0-about) page.

## CODEC

enCoder / Decoder - software that translates data in/out of some format. Particularly: audio data-length compressors / decompressors.

## Dialplan

Set of XML files that parse the call details and decide where to route the call.

## DTMF

**Dual-Tone Multi-Frequency** (DTFM) signaling allows sending tones in voice-frequency bands (in-band). Commonly used over telephone lines, DTMF tones are also commonly called **touch tones**.

## Gateways

From pg. 74 of the FreeSWITCH 1.0.6 book:

> The counterpart to having a user register to your FreeSWITCH server is to have your server register as a user on a remote server. This is accomplished using gateways. A gateway is quite simply an outbound registration to another SIP server. Telephone service providers use very large servers to provide SIP trunks to their subscribers. In FreeSWITCH, we can use a gateway to connect to a SIP provider. We can also use a gateway to connect to another SIP server, such as another FreeSWITCH server or any SIP-compliant IP-PBX.

## Local loop

The connection between a subscriber and the end office of a telecommunications service provider; also referred to as **last mile**.

In business telecommunication environments, the local loop is the physical wiring that connects the company's internal switch (such as a [PBX](#pbx)) to the nearest access point of the telco service provider being used by the company. This is what allows the company access to the [PSTN](#pstn).

## PBX

PBX stands for Private Branch Exchange, which is a private telephone network used within a company or organization. The users of the PBX phone system can communicate internally (within their company) and externally (with the outside world), using different communication channels like [Voice over IP](https://www.3cx.com/pbx/voice-over-ip/), ISDN or analog. A PBX also allows you to have more phones than physical phone lines (PTSN) and allows free calls between users. Additionally, it provides features like transfer calls, voicemail, call recording, interactive voice menus (IVRs) and call queues.

## PRI

Paraphrasing from \<https://searchnetworking.techtarget.com/definition/Primary-Rate-Interface>:

In the Integrated Services Digital Network ([ISDN](https://searchenterprisewan.techtarget.com/definition/ISDN)), there are two levels of service:

* the **[Basic Rate Interface](https://searchnetworking.techtarget.com/definition/Basic-Rate-Interface)** (BRI),  
intended for the home and small enterprise
* the **Primary Rate Interface** (PRI),  
for larger users.

Both rates include a number of **B-channels** and a **D-channel**.

* [B-channel](https://searchnetworking.techtarget.com/definition/B-channel)s carry data, voice, and other services.
* [D-channel](https://searchnetworking.techtarget.com/definition/D-channel)s carry control and signaling information.

### Basic Rate Interface

The **Basic Rate Interface** consists of

* two 64 [Kbps](https://searchnetworking.techtarget.com/definition/Kbps) B-channels and
* one 16 Kbps D-channel.

Thus, a Basic Rate Interface user can have up to 128 Kbps service.

### Primary Rate Interface

The **Primary Rate Interface** channels are carried on

* a [T-carrier system](https://searchtelecom.techtarget.com/definition/T-carrier-system) line (in the U.S., Canada, and Japan) or
* an E-carrier line (in other countries)

and are typically used by medium to large enterprises. The Primary Rate user is hooked up directly to the telephone company central office.

A PRI consists of

* 23 B-channels and
* one 64 Kpbs D-channel
* when using a **T1 line**

or

* 30 B-channels and
* one D-channel
* when using an **E1 line**.

Thus, a **Primary Rate Interface** user can have 

* up to 1.544 [Mbps](https://searchnetworking.techtarget.com/definition/Mbps) service on a T1 line, or
* up to 2.048 Mbps service on an E1 line.

The 23 (or 30) B-channels can be used flexibly and reassigned when necessary to meet special needs such as video conferences. PRI uses the Q.931 protocol over the D-channel. 

For more information, see [ISDN](https://searchenterprisewan.techtarget.com/definition/ISDN). To compare ISDN data rates with other technologies, see [The speed of...](https://whatis.techtarget.com/definition/The-speed-of).

## SIP Profiles

From pg. 79 of the FreeSWITCH 1.0.6 book:

> In the strictest sense of the word, a SIP profile in FreeSWITCH is a User Agent. In practical In practical terms, this means that each SIP profile "listens" on a particular IP address and port number. The internal profile listens on port 5060, and the external profile listens on port 5080\. Not only does the profile listen but it can respond as well. For example, when a phone sends a SIP REGISTER packet to FreeSWITCH (at port 5060), the internal profile "hears" the registration request and acts accordingly. The files in conf/sip\_profiles/ are ones which determine how the profiles behave. Many of the parameters in these profiles are to customize how FreeSWITCH handles various SIP traffic scenarios. In most cases the defaults are reasonable and should work. In other cases, though, you may find that because of the peculiarities in various VoIP phones and vendors, you will need to make adjustments. Lastly, do not let the profile names internal and external be a source of confusion. Each profile is simply a user agent that is streamlined for a specific purpose. The internal profile is optimized to handle telephone registrations and calls between registered phones. The external profile is optimized for outbound gateway connections and several NAT traversal scenarios.

For a deeper discussion of user agents and the concept of a back-to-back user agent (B2BUA) see [wikipedia article about this](http://en.wikipedia.org/wiki/Back-to-back%5Fuser%5Fagent).

## Softphone

A **softphone** is a [software](https://en.wikipedia.org/wiki/Computer%5Fsoftware "Computer software") program for making [telephone](https://en.wikipedia.org/wiki/Telephone "Telephone") calls over the [Internet](https://en.wikipedia.org/wiki/Internet "Internet") using a general purpose computer rather than dedicated hardware. The softphone can be installed on a piece of equipment such as a [desktop](https://en.wikipedia.org/wiki/Desktop%5Fcomputer "Desktop computer"), [mobile device](https://en.wikipedia.org/wiki/Mobile%5Fdevice "Mobile device"), or other computer and allows the user to place and receive calls without requiring an actual telephone set.

## Softswitch

A **softswitch** (_software switch_) is a call switching (see above) node in a [telecommunications network](https://en.wikipedia.org/wiki/Telecommunications%5Fnetwork "Telecommunications network"), 

based not on the specialized switching hardware of the traditional [telephone exchange](https://en.wikipedia.org/wiki/Telephone%5Fexchange "Telephone exchange"), 

but **implemented in software running on a general-purpose computing platform**.

Like its traditional counterparts it connects [telephone calls](https://en.wikipedia.org/wiki/Telephone%5Fcall "Telephone call") between subscribers or other switching systems across a telecommunication network.

VoIP softswitches are subdivided into numerous classes, in analogy to the traditional functions in the [public switched telephone network](https://en.wikipedia.org/wiki/Public%5Fswitched%5Ftelephone%5Fnetwork "Public switched telephone network").

### Classification system

#### Overview

There are five classes of switches, one through five, referring to the size of the area they cover.

* **Class 1** are international gateways
* **Classes 2 through 4** connect cities, towns, and even states
* **Class 5** switches connect to end users

![](https://present5.com/presentation/49bc68ed4c1e9b229019cd91a0dbad4d/image-8.jpg)

![](https://d3i71xaburhd42.cloudfront.net/ecf1d2afd536d7ffd04123ab9053ba756c1ca498/7-Figure1-2-1.png)

#### Class 4 softswitches

**Class 4 softswitches**, analogous with other [Class 4 telephone switches](https://en.wikipedia.org/wiki/Class%5F4%5Ftelephone%5Fswitch "Class 4 telephone switch"), are used for transit VoIP traffic between carriers.

Their main function is to route large volumes of long-distance VoIP calls.

The most important characteristics of Class 4 softswitch are

* protocol support and conversion,
* [transcoding](https://en.wikipedia.org/wiki/Transcoding "Transcoding"),
* [calls per second](https://en.wikipedia.org/wiki/Calls%5Fper%5Fsecond "Calls per second") rate,
* average time of one call routing,
* number of [concurrent calls](https://en.wikipedia.org/wiki/Erlang%5Funit "Erlang unit").

#### Class 5 softswitches

**Class 5 softswitches** are intended to serve subscribers, and connect end user devices.

Class 5 softswitches are characterized by additional services for end-users and corporate clients such as 

* [IP PBX](https://en.wikipedia.org/wiki/IP%5FPBX "IP PBX") features,
* call center services,
* [calling card](https://en.wikipedia.org/wiki/Telephone%5Fcard "Telephone card") platform,
* types of authorization,
* QoS,
* Business Groups,
* and other features similar to other [Class 5 telephone switches](https://en.wikipedia.org/wiki/Class%5F5%5Ftelephone%5Fswitch "Class 5 telephone switch").[\[2\]](https://en.wikipedia.org/wiki/Softswitch#cite%5Fnote-2)

## Telco

A **telco**, also known as a 

* **telephone company**,
* **telephone service provider**,[\[1\]](https://en.wikipedia.org/wiki/Telephone%5Fcompany#cite%5Fnote-1) or
* **telecommunications operator**,

is a kind of **[communications service provider](https://en.wikipedia.org/wiki/Communications%5Fservice%5Fprovider "Communications service provider")** (CSP), more precisely a **[telecommunications service provider](https://en.wikipedia.org/wiki/Telecommunications%5Fservice%5Fprovider "Telecommunications service provider")** (TSP),

that provides [telecommunications](https://en.wikipedia.org/wiki/Telecommunications "Telecommunications") services such as [telephony](https://en.wikipedia.org/wiki/Telephony "Telephony") and [data communications](https://en.wikipedia.org/wiki/Data%5Fcommunication "Data communication") access.

The government agencies are often referred to, primarily in Europe, as PTTs ([postal, telegraph and telephone services](https://en.wikipedia.org/wiki/Postal,%5Ftelegraph%5Fand%5Ftelephone%5Fservice "Postal, telegraph and telephone service")).[\[2\]](https://en.wikipedia.org/wiki/Telephone%5Fcompany#cite%5Fnote-2)

Telephone companies are [common carriers](https://en.wikipedia.org/wiki/Common%5Fcarrier "Common carrier"), and in the [United States](https://en.wikipedia.org/wiki/Local%5Fexchange%5Fcarrier "United States") are also called [local exchange carriers](https://en.wikipedia.org/wiki/Mobile%5Ftelephony "Local exchange carrier"). With the advent of [mobile telephony](https://en.wikipedia.org/wiki/Mobile%5Ftelephony "Mobile telephony"), telephone companies now include [wireless carriers](https://en.wikipedia.org/wiki/Wireless%5Fcarrier "Wireless carrier"), or [mobile network operators](https://en.wikipedia.org/wiki/Mobile%5Fnetwork%5Foperator "Mobile network operator").[\[3\]](https://en.wikipedia.org/wiki/Telephone%5Fcompany#cite%5Fnote-3) Most telephone companies now also function as [internet service providers](https://en.wikipedia.org/wiki/Internet%5Fservice%5Fprovider "Internet service provider") and the distinction between a telephone company and an ISP may disappear completely over time, as the current trend for [supplier convergence](https://en.wikipedia.org/wiki/Supplier%5Fconvergence "Supplier convergence") in the industry continues.[\[4\]](https://en.wikipedia.org/wiki/Telephone%5Fcompany#cite%5Fnote-4)

## URI scheme

From the [Uniform Resource Identifier](https://en.wikipedia.org/wiki/Uniform%5FResource%5FIdentifier) wikipedia page:

> While most URI schemes were originally designed to be used with a particular [protocol](https://en.wikipedia.org/wiki/Protocol%5F%28computing%29 "Protocol (computing)"), and often have the same name, they are semantically different from protocols. ## VoIP

VoIP (voice over IP) is the transmission of voice and multimedia content over Internet Protocol ([IP](https://searchunifiedcommunications.techtarget.com/definition/Internet-Protocol)) networks. VoIP historically referred to using IP to connect private branch exchanges ([PBXs](https://searchunifiedcommunications.techtarget.com/definition/private-branch-exchange)), but the term is now used interchangeably with [IP telephony](https://searchunifiedcommunications.techtarget.com/definition/IP-telephony).

VoIP is enabled by a group of technologies and methodologies used to deliver voice communications over the internet, enterprise [local area networks](https://searchnetworking.techtarget.com/definition/local-area-network-LAN) or [wide area networks](https://searchenterprisewan.techtarget.com/definition/WAN). [VoIP endpoints](https://whatis.techtarget.com/definition/endpoint-device) include dedicated desktop [VoIP phones](https://searchunifiedcommunications.techtarget.com/definition/VoIP-phone), [softphone](https://searchunifiedcommunications.techtarget.com/definition/softphone) applications running on PCs and mobile devices, and [WebRTC](https://searchunifiedcommunications.techtarget.com/definition/WebRTC-Web-Real-Time-Communications)\-enabled browsers.

Read more at \<https://searchunifiedcommunications.techtarget.com/definition/VoIP>
