# Introduction 

 

## About

Welcome to the community documentation wiki for FreeSWITCH, a telephony construction kit for building everything from softphones to full Class 5 switches. You can [embed](https://wiki.freeswitch.org/wiki/Embedding%5FFreeSWITCH "Embedding FreeSWITCH") FreeSWITCH into a soft- (or hard-) phone or OpenWRT router, hook it to PRI circuits through FreeTDM and hardware interfaces such as the Sangoma A100 series, or use it to build an office PBX phone system -- from scratch, or using packaged add-ons like FusionPBX.  FreeSWITCH has a large and active development community, mailing lists, an IRC channel, an annual convention, and weekly videoconference calls hosted on an instance of FreeSWITCH. It's licensed under the MPL, the Mozilla Public License.

Because FreeSWITCH is an engine designed as a platform on which to build many different types of systems, it is inherently modular. Our motto is: "Don't glue the Lego pieces together". FreeSWITCH is a modular application, in which modules can extend the functionality, but the abstraction layer prevents inter-module dependency. The goal is to ensure that one module is not required to load another.


## What Is FreeSWITCH™?

FreeSWITCH™ is an open source communications platform written in C from the ground up. The core is licensed under the [MPL 1.1](http://www.opensource.org/licenses/mozilla1.1.php) but other components have other licensing requirements. Users have many choices of platforms on which to run the software including Windows, Mac OS X, \*BSD, Debian, Ubuntu, and CentOS/Fedora/RHEL.

With a desire not to reinvent the wheel, we've designed FreeSWITCH to take advantage of as many existing software libraries as possible. It has a modular, extensible architecture, with only limited and necessary functionality in its core. Optional modules can be employed (or created) to add virtually any functionality desired by the user.

* Default implementation is for a PBX or Softswitch (actually a back-to-back user agent or B2BUA)  
   * The core ([libfreeswitch](http://wiki.freeswitch.org/wiki/Libfreeswitch "Libfreeswitch")) can be [embedded](http://wiki.freeswitch.org/wiki/Embedding%5FFreeSWITCH "Embedding FreeSWITCH") into almost any app that can use a .so or .dll.  
   * Transform it into a softphone, PBX, soft-switch, conference bridge or a custom telephony application.
* Module system allows you to extend FreeSWITCH easily.  
   * Applications may be written in C/C++, Java, [.NET](http://wiki.freeswitch.org/wiki/Mod%5Fmanaged "Mod managed"), [Javascript/ECMAScript](http://wiki.freeswitch.org/wiki/Javascript "Javascript"), [Python](http://wiki.freeswitch.org/wiki/Mod%5Fpython "Mod python"), [Perl](http://wiki.freeswitch.org/wiki/Perl "Perl"), Ruby, PHP, [Lua](http://wiki.freeswitch.org/wiki/Lua "Lua"), and more!  
   * External systems can receive events from and/or control the switch over a TCP [Event socket](http://wiki.freeswitch.org/wiki/Mod%5Fevent%5Fsocket "Mod event socket") with many language bindings and clients.
* Handles thousands of concurrent channels with media on a standard PC.
* Interoperates with many different products and protocols.  
   * Such as [Asterisk](http://www.asterisk.org/), [Yate](http://yate.null.ro/), or [sipXecs](http://www.sipfoundry.org).  
   * Supports SIP, SCCP, H.323, LDAP, Zeroconf, etc.  
   * With [FreeTDM](http://wiki.freeswitch.org/wiki/FreeTDM "FreeTDM"), a BSD licensed TDM abstraction library, or hardware analog FXO gateways it can interface with the [PSTN](https://confluence.freeswitch.org/display/FREESWITCH/Glossary#Glossary-pstn) as well.
* Supports [Secure RTP](http://wiki.freeswitch.org/wiki/Secure%5FRTP "Secure RTP") (SRTP).
* Please see the complete [Feature list](http://wiki.freeswitch.org/wiki/Features "Features") for more features.
