
# TTML 

 

[Timed Text Markup Language](https://en.wikipedia.org/wiki/Timed%5FText%5FMarkup%5FLanguage) "_was designed to be used for the purpose of authoring, transcoding or exchanging timed text information presently in use primarily for [subtitling and captioning](https://en.wikipedia.org/wiki/Subtitle%5F%28captioning%29 "Subtitle (captioning)")_ _functions_", but apparently FreeSWITCH uses them to compose sequences of tones. 

`fur_elise.ttml` comes with a vanilla FreeSWITCH installation, and it is a sequence of FreeSWITCH tones:

**Snippet from fur\_elise.ttml**

```xml
%(167, 0, 659)
```

## Usage

Taking the dialplan example from [XML Dialplan](../../../../Dialplan/XML-Dialplan/index.mdx#about):

| &lt;extension name\="US-Domestic"\> &lt;condition field\="destination\_number" expression\="\_(NXXXXXXXXX)"\> &lt;action application\="bridge" data\="sofia/internal/$1@[example.com](http://example.com)"/> &lt;/condition\>&lt;/extension\> &lt;extension name\="star-code-using-escape"\> &lt;condition field\="destination\_number" expression\="\_(\\\*XX)(.)"\> &lt;action application\="log" data\="ERR captured $1 \~\~\~ $2"/> &lt;action application\="answer"/> &lt;action application\="playback" data\="tone\_stream://path=${base\_dir}/conf/tetris.ttml;loops=10"/> &lt;/condition\>&lt;/extension\> |
